Using the desktop client

The event save file connects to the Internet by utilizing the GBA Link Cable - more precisely, the JoyBus protocol. Turns out, two of the most popular Game Boy Advance emulators, mGBA and VBA-M, support JoyBus transmissions over TCP/IP. Thanks to this, it's possible to connect them to the event servers by using a dedicated client application. Here are some instructions to help you set this up.

Download everything!

We begin by downloading a bunch of stuff:

  • The emulator, obviously! You can either use endrift's excellent mGBA emulator (recommended), or the latest version of VBA-M. Normal VBA or VBA-Link won't suffice - please quit using these ancient emulators! If you have any doubts, just download mGBA.
  • The event save file. It can be downloaded at the itch.io project page - look for the SAV file tagged with the latest version.
  • The desktop client executable. Your emulator won't be able to connect to a remote service by itself, since link cable functionalities are intended mostly for local use. To help contact the event servers, a local proxy needs to be created. The emulator will send requests to a server launched on your machine, which will forward them to the Internet. Just like before, you can find the Fools2020DesktopClient.exe on the itch.io project page
  • A clean ROM image of Pokémon Emerald (U). You're on your own on that one!

Set things up!

Start by opening your emulator, loading the ROM, and importing the SAV file:

  • In mGBA, use the "File » Load alternate save game" option.
  • In VBA-M, use the "File » Import » Battery file" option.

Then, launch the client application (Fools2020DesktopClient.exe). If you're running Linux - no worries, the application works under Wine without problems. Log in using your credentials from the event site. Then, connect to the local server:

  • Regardless of the emulator you're using - load the save file first, and play until you see the "Welcome to TheZZAZZGlitch's Fools2022..." message. This is an important step.
  • Then, if you're using mGBA:
    • Select "File » Connect to Dolphin".
    • Make sure "Local computer" is selected.
    • Click "Connect".
  • Or if you're using VBA-M:
    • Select "Options » Link » Type » GameCube".
    • Afterwards, select "Options » Link » Start network link".
    • Select "Client". In the server field, type in localhost.
    • Click "Connect".

Notes and troubleshooting

  • Do not use savestates, however tempting. Savestating might cause desynchronization between server and client states, leading to connection errors, or even loss of progress. Or even worse, some quests might become unfinishable. Just don't use savestates, trust me.
  • Users are disconnected after 30 minutes of inactivity. If you exceed this limit, you'll have to restart the client and log in again.
  • Sometimes, the emulator might just disconnect on its own - such is the nature of networking. If you're getting connection errors, reconnecting the emulator is the first thing you should try.
